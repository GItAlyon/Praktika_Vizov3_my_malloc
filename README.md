# Аллокатор памяти my_malloc

## Описание проекта

Данный проект представляет собой **полноценную реализацию аллокатора памяти** на языке C, включающую все основные функции управления динамической памятью: `malloc`, `free`, `realloc` и `calloc`. 

Аллокатор разработан в учебных целях для демонстрации внутреннего устройства систем управления памятью. Он поддерживает **Best-Fit** и **First-Fit** стратегии поиска, выполняет **слияние** и **разделение** блоков, а также содержит встроенные средства **отладки** и **защиты** от ошибок.


## Основные возможности

### 1. Управление памятью
- **Выделение памяти** (`my_malloc`) с выравниванием до 16 байт
- **Освобождение памяти** (`my_free`) с проверкой на двойное освобождение
- **Изменение размера** (`my_realloc`) с оптимизацией на месте
- **Выделение с обнулением** (`my_calloc`)

### 2. Управление блоками
- **Заголовки блоков** с метаданными (размер, магическое число, canary)
- **Двусвязные списки** свободных и занятых блоков
- **Слияние** соседних свободных блоков (coalescing)
- **Разделение** больших блоков (splitting)

### 3. Стратегии поиска
- **Best-Fit** - поиск наименьшего подходящего блока (меньше фрагментации)
- **First-Fit** - поиск первого подходящего блока (быстрее)

### 4. Безопасность и отладка
- **Canary-значения** для обнаружения переполнения буфера
- **Магические числа** для проверки целостности
- **Детектирование двойного освобождения**
- **Валидация указателей** перед операциями
- **Режим отладки** с пошаговым логированием
- **Проверка целостности** всей кучи

### 5. Статистика
- Сбор полной статистики работы аллокатора
- Отслеживание пикового использования памяти
- Подсчёт операций слияния и разделения

### 6. Многоплатформенность
- **Windows** - использование `VirtualAlloc`
- **Linux/Unix** - использование `sbrk`

## Безопасность
Аллокатор включает несколько уровней защиты:

1. Canary-значения
```c
// Каждый блок содержит сторожевой байт
block->canary = 0x5A5A5A5A;

// При переполнении буфера canary изменяется
if (!check_canary(block)) {
    // Обнаружено переполнение!
}
```
2. Магические числа
```c
// Каждый блок содержит уникальную метку
block->magic = MAGIC_NUMBER;  // 0xDEADBEEF

// Проверка при каждой операции
if (block->magic != MAGIC_NUMBER) {
    // Блок повреждён!
}
```
3. Валидация указателей
```c
// Проверка, что указатель в пределах кучи
if ((uintptr_t)block < (uintptr_t)heap_start) {
    // Указатель вне кучи!
}
```
4. Защита от двойного освобождения
```c
if (block->free) {
    fprintf(stderr, "ПРЕДУПРЕЖДЕНИЕ: Двойное освобождение!\n");
    return;
}
```

## Производительность
Асимптотическая сложность
| Операция | Сложность |	Пояснение |
|----------|-----------|------------|
| malloc (Best-Fit) |	O(n) | Обход всех свободных блоков |
| malloc (First-Fit) | O(n) | Обход до первого подходящего |
| free | O(n) | Обход для слияния соседей |
| realloc | O(n) | + копирование данных |

## Оптимизации
- Выравнивание до 16 байт - для SSE/AVX инструкций

- Запрос больших чанков - уменьшение системных вызовов

- Слияние блоков - уменьшение фрагментации

- Разделение блоков - эффективное использование памяти

- Кэширование - свободные блоки хранятся для переиспользования

## Известные ограничения
- Не потокобезопасен - нет синхронизации для многопоточности

- Не возвращает память ОС - при освобождении блоки остаются в куче

- Ограниченная отказоустойчивость - при повреждении кучи может упасть

- Линейный поиск - не использует битовые карты или хеш-таблицы
